## GitHub Pages 部署修复

**问题**: https://leappower.github.io/KitchenYuKoLi/ 访问 404

**根因**:
1. SSG 生成的 `index.html` 使用 `location.href = '/home/'` 重定向，但 GitHub Pages 子路径部署需要 `/KitchenYuKoLi/home/`
2. SSG 生成的 `404.html` 中资源路径是 `/assets/...`，没有 base path 前缀
3. release.js 的 `patchDistBasePath` 正则无法处理所有情况

**修复**:
1. `build-ssg.js` 新增 `--base-path` 参数，支持为所有 HTML 路径添加前缀
2. 新增 `patchHtmlPaths()` 函数统一处理 src/href 属性和内联 JS 路径
3. 修复 canonical URL 和 og:url 的双斜杠拼接问题
4. `release.js` Step 8 改为：标准构建 → 用 `--base-path` 重新运行 SSG

**验证**:
```bash
npm run build:pack
node scripts/build-ssg.js --clean --base-path=/KitchenYuKoLi
grep 'src="' dist/home/index.html  # 应显示 /KitchenYuKoLi/assets/...
```

**部署**:
```bash
node scripts/release.js --skip-feishu --skip-translate --skip-download --gh-pages
```

## GitHub Pages 404 修复 (v0.0.3)

**问题**: 资源路径仍然是 `/assets/...` 而不是 `/KitchenYuKoLi/assets/...`

**根因**: `build-ssg.js` 中的 `patchHtmlPaths()` 函数使用了硬编码的 `'KitchenYuKoLi'` 字符串，而不是动态的 `bp` 变量

**修复**:
- 将硬编码的正则表达式改为使用 `RegExp` 构造函数动态生成
- 从 `BASE_PATH` 动态提取 `bpName` 用于 negative lookahead

**发布**:
- 版本：v0.0.2 → v0.0.3
- 命令：`node scripts/release.js --skip-feishu --skip-translate --skip-download --gh-pages`
- 访问地址：https://leappower.github.io/KitchenYuKoLi/

## GitHub Pages JS 资源路径修复 (v0.0.4)

**问题**: JS 动态加载的资源（语言文件、路由页面）仍然是 `/assets/...` 而不是 `/KitchenYuKoLi/assets/...`

**根因**: 
1. `generateRootIndex()` 没有调用 `patchHtmlPaths()`
2. `translations.js` 和 `spa-router.js` 使用硬编码的绝对路径

**修复**:
- `build-ssg.js`: 在 `generateRootIndex()` 中添加 `patchHtmlPaths()` 调用
- `build-ssg.js`: 在 HTML 中注入 `window.BASE_PATH` 全局变量
- `translations.js`: 使用 `window.BASE_PATH` 加载语言文件
- `spa-router.js`: 使用 `window.BASE_PATH` 加载路由页面

**发布**:
- 版本：v0.0.3 → v0.0.4
- 命令：`node scripts/release.js --skip-feishu --skip-translate --skip-download --gh-pages`
